# Flask Project Patterns

Standard structure, conventions, and hard requirements for all Flask projects in this portfolio.
Compatible with [flask-deploy](https://github.com/massyn/flask-deploy) — a curl-able bash script
that deploys Flask apps via Gunicorn + Nginx + systemd.

---

## Hard requirements

- **`DATABASE_URL` must be set explicitly in the environment.** No default is provided. If missing,
  the app raises `RuntimeError` at startup and refuses to run. This is enforced in `config.py`
  before the `Config` class is defined.
- For SQLite in production, `DATABASE_URL` must point to `/data/<slug>/app.db` so it survives
  redeployment. The deploy script creates `/data/<slug>/` and never wipes it.
- `SECRET_KEY` defaults to a dev value but must be replaced in production.
- `requirements.txt` must list `gunicorn` — the deploy script validates this.

---

## flask-deploy contract

### `run.py`

Must be at the repository root. Must expose a module-level `app` variable (Gunicorn is invoked as
`gunicorn run:app` — filename and variable name are fixed). Must include an `if __name__ == '__main__'`
block using env vars:

```python
import os
from pathlib import Path
from dotenv import load_dotenv

_env_file = Path('.env_dev') if Path('.env_dev').exists() else Path('.env')
load_dotenv(_env_file)

from app import create_app

app = create_app()

if __name__ == '__main__':
    app.run(
        host=os.environ.get('FLASK_HOST', '0.0.0.0'),
        port=int(os.environ.get('FLASK_PORT', 5000)),
        debug=os.environ.get('FLASK_DEBUG', 'False').lower() == 'true',
    )
```

### `requirements.txt`

Must be at the repository root. Must include `gunicorn`.

### `.gitignore`

Must exclude at minimum: `venv/`, `gunicorn_config.py`, `.env_dev`.

- `.env_dev` is a local developer override — git-ignored so the prod server never picks up dev behaviour.
- `.env` is committed to the repo and deployed to `/opt/<slug>/` alongside the code. **Security note:**
  this is a deliberate pragmatic trade-off for solo operators — use a private repo. For team or hardened
  production environments, move secrets out of git.

`gunicorn_config.py` is generated by the deploy script on every run — do not commit it.

## Deployment directory layout

| Path | Purpose |
|------|---------|
| `/opt/<slug>/` | Git clone — includes `.env`; disposable except for `.env` |
| `/data/<slug>/` | Stateful data: SQLite databases — **back this up** |
| `/var/log/<slug>/` | Logs: `access.log`, `error.log`, `nginx.access.log`, `nginx.error.log` |

## Config pattern (`config.py`)

```python
import os
from datetime import timedelta

_database_url = os.environ.get('DATABASE_URL', '')
if not _database_url:
    raise RuntimeError('DATABASE_URL is required — set it in your .env file. No default is provided.')

class Config:
    SECRET_KEY = os.environ.get('SECRET_KEY', 'dev-secret-please-change')
    DATABASE_URL = _database_url
    GOOGLE_CLIENT_ID = os.environ.get('GOOGLE_CLIENT_ID', '')
    GOOGLE_CLIENT_SECRET = os.environ.get('GOOGLE_CLIENT_SECRET', '')
    TEST_USER = os.environ.get('TEST_USER', '')
    PERMANENT_SESSION_LIFETIME = timedelta(days=30)
    FORCE_HTTPS = os.environ.get('FORCE_HTTPS', 'false').lower() == 'true'
    VERSION = os.environ.get('VERSION', '')
```

Note: `BASE_URL` has been removed. Use the `CF-Visitor` + `FORCE_HTTPS` pattern for OAuth redirect URIs.

## Dev port assignment convention

Flask's dev server does not detect port conflicts gracefully. Each app is assigned a fixed dev port.
Set it in `.env_dev` — never rely on the default 5000.

**Port registry:**

| Port | App |
|------|-----|
| 5000 | justride |
| 5001 | findings |
| 5002 | citadel |
| 5003 | secrets |
| 5008 | soapbox |

In each app's `.env_dev`:
```
FLASK_PORT=5000   # replace with this app's assigned port
```

In production, flask-deploy auto-assigns the next available port — `FLASK_PORT` in `.env` is not
needed and should be omitted.

## Admin UI conventions

### List views

- No edit or delete buttons on list rows — they add visual noise and invite accidental clicks.
- The **entire row is clickable** and navigates to the detail/edit screen. Use Bootstrap's
  `table-hover` and a cursor pointer on `<tr>` elements.
- Delete is only accessible from the edit/detail screen — never from the list.
- All tables must be **paginated** — never render unbounded lists.
- All table column headers must be **sortable** — clicking a header sorts ascending; clicking again
  sorts descending. Indicate sort direction with an icon.

```html
<!-- Clickable row pattern -->
<tr class="table-hover" style="cursor:pointer"
    onclick="window.location='{{ url_for('namespace.detail', id=item.id) }}'">
  <td>{{ item.name }}</td>
  <td>{{ item.created_on }}</td>
</tr>
```

### Pagination

- Pass `page` (int, default 1) and `per_page` (int, default 25) as query params.
- Route fetches `LIMIT per_page OFFSET (page-1)*per_page` from the DB.
- Render Bootstrap pagination component; show current page and total pages.
- Never paginate client-side — always paginate in SQL.

### Sortable headers

- Pass `sort` (column name) and `direction` (`asc`/`desc`) as query params.
- Whitelist allowed sort columns in the route — never pass raw query params to SQL.
- Render header links that toggle direction on re-click.

### Delete confirmation

- Delete button on edit screen only.
- Always confirm before deleting — use a Bootstrap modal, not `window.confirm()`.
- POST to a dedicated `/delete` route — never delete on GET.

## Session persistence policy

The default is **persistent sessions** — sessions survive browser restarts. This is appropriate for
general-purpose apps where UX matters more than security posture.

**Exception: sensitive applications must use non-persistent sessions.** Apps that manage credentials,
secrets, security configuration, or other sensitive data must not use persistent sessions.

| App type | Session policy |
|----------|---------------|
| General (justride, findings, etc.) | Persistent — `session.permanent = True`, 30-day lifetime |
| Sensitive (secrets, security tools) | Non-persistent — do NOT set `session.permanent = True` |

```python
# General apps — persistent sessions
session.permanent = True
session['user'] = { ... }

# Sensitive apps — non-persistent, browser-close expiry
# Do NOT set session.permanent
session['user'] = { ... }
```

## Version footer

Set `VERSION` in `.env` to display an inconspicuous version indicator in the footer of every page.

- If `VERSION` is unset or empty, the footer renders nothing.
- If set, it shows as a small, muted footer label.

In `config.py`:
```python
VERSION = os.environ.get('VERSION', '')
```

Inject into templates via a context processor in the app factory:

```python
@app.context_processor
def inject_version():
    return dict(version=app.config.get('VERSION', ''))
```

In `base.html`:

```html
{% if version %}
<footer class="text-center text-muted small py-2">
    {{ version }}
</footer>
{% endif %}
```

## Reverse proxy / OAuth redirect URI

Flask runs behind Nginx in production. Without intervention, `url_for(..., _external=True)` produces
`http://` URLs, which breaks Google OAuth.

**Do not use BASE_URL** — it hardcodes the domain.
**Do not rely on ProxyFix alone** — it only works if Nginx forwards `X-Forwarded-Proto`.

Instead, use a helper that checks in order:

1. **Cloudflare `CF-Visitor` header** — present when request came in over HTTPS via Cloudflare.
2. **`FORCE_HTTPS=true`** — explicit override for non-Cloudflare deployments.
3. **Flask native** — falls back to whatever scheme Flask sees. Correct for local dev.

```python
def get_oauth_redirect_uri() -> str:
    cf_visitor = request.headers.get('CF-Visitor', '')
    if '"scheme":"https"' in cf_visitor:
        return url_for('auth.callback', _external=True, _scheme='https')
    if current_app.config.get('FORCE_HTTPS'):
        return url_for('auth.callback', _external=True, _scheme='https')
    return url_for('auth.callback', _external=True)
```

## Remote IP resolution

Never use `request.remote_addr` directly — it reflects the upstream proxy, not the real client.

```python
def get_remote_ip() -> str:
    return (
        request.headers.get('CF-Connecting-IP')
        or request.headers.get('X-Real-IP')
        or request.remote_addr
        or ''
    )
```

Order: Cloudflare → Nginx → socket (local dev only). Never trust `X-Forwarded-For` — it is
user-controlled and can be spoofed.

## Dotenv loading pattern

Load `.env_dev` when it exists; fall back to `.env` otherwise.

In `run.py` (before any app imports):

```python
from pathlib import Path
from dotenv import load_dotenv

_env_file = Path('.env_dev') if Path('.env_dev').exists() else Path('.env')
load_dotenv(_env_file)
```

- `.env` — production secrets; committed to git; lives in `/opt/<slug>/` on the server.
- `.env_dev` — local developer overrides; takes priority when present; git-ignored.
- `.env.example` — committed; documents all required variables.

**Critical:** `load_dotenv()` must be called before any `from config import Config` statement.

## Project structure

```
my-app/
  run.py                  # Entry point — exposes `app`; required by flask-deploy
  config.py               # Config class; all values from env vars
  requirements.txt        # Must include gunicorn
  .env                    # Production secrets — committed to git (private repo assumed)
  .env_dev                # Local dev overrides — git-ignored; takes priority over .env
  .env.example            # Committed; documents all required vars
  .gitignore              # Must exclude venv/, gunicorn_config.py, .env_dev
  static/
  templates/
    base.html             # Base layout; all pages extend this
    sql/                  # All SQL lives here as Jinja .sql files
  myapp/                  # App package — named after the project
    __init__.py           # App factory
    db.py                 # DB connection + SQL template rendering
    main.py               # Main blueprint (public routes)
    auth/
      __init__.py         # Blueprint + OAuth routes
      service.py          # User lookup/creation + get_remote_ip + get_oauth_redirect_uri
      decorators.py       # login_required etc.
```

## App package layout

The app code lives in a subdirectory named after the project. The root contains only: `run.py`,
`config.py`, `requirements.txt`, and the `.env*` files. `templates/` and `static/` stay at the root.

Because `templates/` and `static/` are at the root, the app factory must explicitly point Flask at them:

```python
import os

def create_app() -> Flask:
    root = os.path.dirname(os.path.dirname(__file__))
    app = Flask(
        __name__,
        template_folder=os.path.join(root, 'templates'),
        static_folder=os.path.join(root, 'static'),
    )
```

## Blueprints

- One blueprint per feature area (auth, main, profile, …).
- Registered in the app factory.
- Route handlers are thin: validate → call service → `render_template()`. No business logic in routes.

---

## SQL pattern

- All queries in `templates/sql/` as Jinja2 `.sql` files (e.g. `user_insert.sql`).
- Templates receive `dialect` (`"sqlite"` or `"postgres"`) for conditional blocks.
- Parameter placeholders: `?` for SQLite, `%s` for psycopg3 — emitted via the template.
- Never use f-strings or `.format()` to inject values; always use parameterised queries.
- One file per logical query.

---

## Templating

- `base.html` defines layout and navigation; all page templates extend it.
- Bootstrap 5 loaded via CDN.
- `url_for()` for all internal links — never hardcode paths.
- Flash messages via `flask.flash()` rendered in `base.html`.
- Templates are presentation only — no business-rule conditionals, no data transformation.

### Navbar

Always use `sticky-top` on the navbar. Do not use `fixed-top` (requires manual body padding).

```html
<nav class="navbar navbar-expand-lg navbar-dark bg-dark sticky-top">
```

## CSRF protection

`flask-wtf` provides global CSRF protection. Add it to `requirements.txt` and wire it in the app factory:

```python
from flask_wtf.csrf import CSRFProtect

csrf = CSRFProtect()

def create_app() -> Flask:
    ...
    csrf.init_app(app)
```

Every POST form in a template must include the token:

```html
<input type="hidden" name="csrf_token" value="{{ csrf_token() }}">
```

AJAX requests must send the token in a header:

```javascript
headers: { 'X-CSRFToken': document.querySelector('[name=csrf_token]').value }
```

To exempt a specific route (e.g. a webhook endpoint):

```python
from app import csrf

@csrf.exempt
@bp.route('/webhook', methods=['POST'])
def webhook():
    ...
```

## Security headers (Flask-Talisman)

`flask-talisman` sets security headers automatically: HSTS, CSP, X-Frame-Options,
X-Content-Type-Options, and Referrer-Policy. Add it to `requirements.txt` and initialise in the
app factory:

```python
from flask_talisman import Talisman

def create_app() -> Flask:
    ...
    csp = {
        'default-src': ["'self'"],
        'style-src': [
            "'self'",
            'https://cdn.jsdelivr.net',   # Bootstrap CSS
        ],
        'script-src': [
            "'self'",
            'https://cdn.jsdelivr.net',   # Bootstrap JS / Popper
            # 'https://static.cloudflareinsights.com',  # add if using Cloudflare analytics
        ],
        'connect-src': [
            "'self'",
            'https://cdn.jsdelivr.net',   # CDN source maps are fetched as network requests
        ],
        'font-src': ["'self'", 'data:'],
        'img-src': ["'self'", 'data:'],
    }
    Talisman(app, content_security_policy=csp)
```

**CSP is strict by default — overly strict settings will silently break Bootstrap CDN and any inline
scripts.** Always whitelist CDN origins explicitly. If you load additional third-party assets (fonts,
icons, analytics), add their origins to the relevant directive.

If a route must serve an inline script (e.g. a Jinja-rendered block), add a nonce rather than
relaxing CSP globally:

```python
# In the route
from flask_talisman import talisman

@bp.route('/chart')
@talisman(content_security_policy={
    'script-src': ["'self'", "'unsafe-inline'"],
})
def chart():
    ...
```

## Google OAuth auth bypass for testing (`TEST_USER`)

Set `TEST_USER=<name>` in `.env_dev` to skip the Google OAuth round-trip during development:

- When `TEST_USER` is set, `/auth/login` redirects to `/auth/test-callback` instead of Google.
- The test callback uses `test:<name>` as the fake Google ID.
- When `TEST_USER` is unset the code path is completely untouched.

## Local development

```bash
python -m venv venv
source venv/bin/activate   # Linux/macOS
venv\Scripts\activate      # Windows
pip install -r requirements.txt
python ./run.py
```

Copy `.env.example` to `.env_dev` and fill in your local values. Set `FLASK_PORT` to this app's
assigned port. Do not set `FORCE_HTTPS` in `.env_dev`. The app loads `.env_dev` automatically
when present. Gunicorn is production-only.

## Dry-run validation (flask-deploy checks)

```
[✓] run.py found
[✓] requirements.txt found
[✓] gunicorn in requirements.txt
[✓] app object found in run.py
[!] .env missing
[!] DATABASE_URL not pointing to /data/<slug>/ — if your db gets hosed on redeploy, you had fair warning
```

`[✗]` items cause a non-zero exit. `[!]` items are warnings that do not block the run.
